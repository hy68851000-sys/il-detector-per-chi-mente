# Lie Detector MVP 需求文档

## 1. 项目概述

**项目名称**：Il Detector per Chi Mente（说谎检测器）

**项目目标**：开发一个基于 AI 的文本分析工具，帮助用户评估文本内容的可信度，识别潜在的欺骗性语言模式。

**目标用户**：
- 需要验证信息真实性的个人用户
- HR 面试官（辅助工具）
- 内容审核人员
- 教育工作者（教学演示）

**核心价值**：通过 AI 分析提供客观的可信度评估，帮助用户做出更明智的判断。

---

## 2. MVP 功能范围

### 2.1 核心功能（必须实现）

#### ✅ 文本分析
- 用户输入待分析文本（最少 50 字，最多 2000 字）
- 点击"分析"按钮触发 AI 分析
- 显示分析结果

#### ✅ 分析结果展示
**输出内容：**
1. **可信度评分**（0-100 分）
   - 0-30：高度可疑
   - 31-60：需要进一步验证
   - 61-85：基本可信
   - 86-100：高度可信

2. **关键指标**（3-5 条）
   - 逻辑矛盾点
   - 情绪异常波动
   - 模糊/回避性用词
   - 过度细节或过度简略
   - 时间线不一致

3. **追问建议**（2-3 个问题）
   - 针对可疑点的具体问题
   - 帮助用户深入验证

#### ✅ 基础界面
- 简洁的单页应用
- 意大利语界面（主要）
- 响应式设计（支持手机/平板/桌面）

---

### 2.2 技术实现

**前端：**
- HTML + CSS + JavaScript（原生）
- 单页应用，无需框架
- 渐变紫色主题

**后端：**
- Cloudflare Workers + Hono 框架
- OpenAI API (gpt-4o-mini)
- RESTful API 设计

**部署：**
- 后端：Cloudflare Workers
- 前端：Cloudflare Pages
- 全球 CDN 加速

## 3. 用户流程

```
1. 用户访问网站
   ↓
2. 在文本框输入待分析内容
   ↓
3. 点击"Analizza"按钮
   ↓
4. 显示加载状态（"Analisi in corso..."）
   ↓
5. 展示分析结果：
   - 可信度评分
   - 关键指标列表
   - 追问建议
   ↓
6. 用户可继续输入新文本分析
```

## 4. API 设计

### 4.1 分析接口

**端点**：`POST /analyze`

**请求体**：
```json
{
  "text": "待分析的文本内容"
}
```

**响应体**：
```json
{
  "success": true,
  "analysis": {
    "credibility_score": 75,
    "indicators": [
      "使用了多个模糊限定词（'可能'、'大概'）",
      "情绪表达过于平淡，缺乏自然波动",
      "时间描述存在细微矛盾"
    ],
    "questions": [
      "能否提供更具体的时间和地点？",
      "当时还有其他人在场吗？"
    ]
  }
}
```

**错误响应**：
```json
{
  "error": "Text is required"
}
```

## 5. 界面设计要求

### 5.1 布局
- 居中卡片式设计
- 最大宽度 600px
- 白色卡片 + 渐变紫色背景

### 5.2 组件
1. **标题区**
   - 主标题：🔍 Detector per Chi Mente
   - 副标题：Analizza il testo con l'intelligenza artificiale

2. **输入区**
   - 多行文本框（最小高度 150px）
   - 占位符：Inserisci il testo da analizzare...

3. **操作区**
   - 分析按钮（全宽，紫色）

4. **结果区**（初始隐藏）
   - 可信度评分（大字号，紫色）
   - 指标列表（带项目符号）
   - 追问建议（带项目符号）

5. **加载状态**
   - 居中显示"Analisi in corso..."

## 6. 非功能需求

### 6.1 性能
- 页面加载时间 < 2 秒
- API 响应时间 < 5 秒
- 支持并发 100 用户

### 6.2 安全
- HTTPS 加密传输
- API Key 环境变量存储
- CORS 跨域保护
- 输入长度限制（防止滥用）

### 6.3 可用性
- 响应式设计（手机/平板/桌面）
- 清晰的错误提示
- 加载状态反馈

## 10. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| OpenAI API 成本过高 | 高 | 设置请求频率限制，考虑替代模型 |
| AI 分析不准确 | 高 | 优化 Prompt，增加人工校验 |
| 用户滥用 API | 中 | 添加 Rate Limiting，验证码 |
| Cloudflare 免费额度不足 | 中 | 监控用量，准备升级计划 |
